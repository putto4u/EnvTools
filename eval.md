\*\*`eval`\*\*은 프로그래밍에서 사용되는 함수 또는 명령어로, **문자열로 된 코드를 실행 시간에 해석하고 실행**하는 기능을 수행합니다.

이는 문자열을 일반적인 데이터(텍스트)로 처리하지 않고, 컴파일되거나 인터프리터에 의해 실행될 수 있는 \*\*실제 코드(명령)\*\*로 취급하게 만듭니다.

-----

## 1\. 🐍 Python에서의 `eval()` 함수

Python에서 `eval()` 함수는 문자열로 된 \*\*Python 표현식(Expression)\*\*을 인수로 받아 실행하고, 그 결과를 반환합니다.

### A. 구문

```python
eval(expression, globals=None, locals=None)
```

### B. 주요 용도

  * **동적 코드 실행:** 사용자 입력이나 파일에서 읽어 들인 문자열을 코드로 실행해야 할 때 사용됩니다. (예: 계산기 프로그램에서 문자열 수식 계산)

  * **예시:**

    ```python
    x = 10
    expression = "x * 2 + 5"
    result = eval(expression)
    print(result)  # 출력: 25
    ```

### C. ⚠️ 보안 위험 강조 (실전 팁)

Python의 `eval()`은 **매우 위험**할 수 있습니다. 사용자로부터 입력받은 문자열에 악의적인 코드(예: 시스템 파일 삭제 명령)가 포함되어 있다면, `eval()`은 그 코드를 그대로 실행하여 **보안 취약점**을 야기합니다.

> **전문가 조언:** 사용자 입력에 `eval()`을 사용하는 것을 **절대 피해야** 하며, 불가피하게 사용해야 할 경우 반드시 입력 문자열을 철저히 검증하고 제한된 환경($\text{globals}, \text{locals}$ 매개변수 사용)에서만 실행해야 합니다.

## 2\. 🖥️ 쉘(Shell)에서의 `eval` 명령어

Bash와 같은 유닉스 계열 쉘에서도 `eval` 명령어가 존재하며, 이는 **문자열을 쉘 명령어로 실행**하는 역할을 합니다.

### A. 동작 원리

1.  \*\*`eval`\*\*에 전달된 문자열을 명령어로 인식하고, 한 번 더 \*\*쉘 확장(Shell Expansion)\*\*을 수행합니다.
2.  확장된 결과를 쉘 명령어로 **실행**합니다.

### B. 주요 용도

  * **환경 설정 스크립트:** 앞서 `pyenv` 설치 시 보셨던 `eval "$(pyenv init --path)"` 구문이 대표적인 예입니다.

      * $\text{pyenv init --path}$는 쉘 명령어로 실행되어 환경 설정 명령어를 **문자열**로 출력합니다.
      * **`$(...)`** 구문이 그 문자열을 캡처하여 \*\*`eval`\*\*로 전달합니다.
      * \*\*`eval`\*\*이 이 문자열(환경 설정 명령어)을 받아서 **현재 쉘 환경에서 실행**하여 $\text{pyenv}$를 활성화합니다.

  * **예시:**

    ```bash
    command_str="echo \$USER"  # $USER는 아직 확장되지 않은 상태로 문자열에 저장됨
    echo $command_str         # 출력: $USER (변수 확장 안 됨)

    eval $command_str         # 출력: 현재 사용자 이름 (eval에 의해 $USER가 확장되어 실행됨)
    ```

### C. ⚠️ 보안 및 안정성

쉘의 `eval` 역시 **보안 위험**이 크며, **스크립트의 안정성**을 떨어뜨릴 수 있습니다. `pyenv`와 같이 **신뢰할 수 있는 소스**에서 생성된 문자열에 대해서만 사용하는 것이 일반적입니다.
